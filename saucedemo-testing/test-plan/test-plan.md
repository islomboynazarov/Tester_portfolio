**1. Objective**

    The objective of this test plan is to verify the core functional behavior of the Sauce Demo e-commerce website, ensuring that critical user flows work as expected and major defects are identified before release.

**2. Application Under Test (AUT)**

    Application Name: Sauce Demo
    URL: https://www.saucedemo.com/
    Application Type: Web-based e-commerce application

**3. Scope of Testing**

 3.1 In-Scope Features
 
    The following features will be tested:
    Login functionality
    Product listing
    Product sorting
    Shopping cart 
    Checkout process

  3.2 Out-of-Scope Features

    Performance testing 
    Security testing
    Accessibility testing 
    Mobile testing 
    Automation testing
    Backend/API testing

**4. Test Types**

    Functional testing
    Exploratory testing
    Regression testing
    UI validation (basic)

**5. Test Approach**

    Testing will be conducted manually by executing predefined test scenarios and test cases, supplemented with exploratory testing to identify unexpected defects. 
    Critical features such as Login and Checkout will be tested in more detail using structured test cases.
    Defects will be logged with clear reproduction steps and supporting evidence.

**6. Test Environment**

    Operating System: Windows 10
    Browser: Brave (latest version)
    Internet Connection: Stable broadband
    Test Data: Predefined test credentials provided by the application

**7. Entry Criteria**

    Testing will begin when:
    The Sauce Demo website is accessible
    Test environment is available
    Test scenarios and test cases are prepared

**8. Exit Criteria**

    Testing will be considered complete when:
    All planned test scenarios are executed
    Critical test cases are executed
    Identified defects are documented
    A test execution report is prepared

**9. Deliverables**

    Test Plan
    Test Scenarios / Checklists
    Test Cases
    Bug Reports
    Test Execution Report

**10. Risks and Assumptions**

    Risks

    Test data is limited to demo credentials
    Some defects may be intentionally present
    Environment behavior may change unexpectedly

    Assumptions**
 
    The application is stable during testing
    Demo users represent real user behavior

**11. Test Responsibility**

  Tester: Manual QA Engineer (Portfolio Project)

12. Approval

  This test plan is prepared for educational and portfolio purposes to demonstrate manual testing skills and QA documentation practices.
