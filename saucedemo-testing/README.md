## QA Workflow Covered
This portfolio follows the standard manual QA process:

1. Test Planning  
2. Test Scenario Design  
3. Test Case Development  
4. Test Execution  
5. Defect Reporting  
6. Test Summary and Results Analysis  

Each step is documented and traceable.


Each folder contains its own README explaining its role in the testing lifecycle.

---

## Key Artifacts Description

### Test Plan
Defines the testing scope, objectives, approach, and assumptions for the application under test.

### Test Scenarios
High-level testing conditions derived from the test plan, focusing on **what needs to be tested**.

### Test Cases
Detailed, step-by-step instructions created from test scenarios, including test data and expected results.

### Test Execution
Execution results recorded in an Excel file, showing pass/fail status and references to reported defects.

### Bug Reports
Individually documented defects with clear reproduction steps, expected vs actual results, severity, and priority.

### Test Summary
A consolidated report describing overall test coverage, execution results, and quality assessment of the application.
